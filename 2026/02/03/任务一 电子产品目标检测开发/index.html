任务一：电子产品目标检测数据集标注

一、任务背景与目标

在目标检测任务中，高质量的标注数据集是模型训练的基础。本次任务主要完成显示屏、键盘、鼠标、手柄的图像采集、LabelMe标注工具使用、多边形标注、标签规范、文件导出与格式检查，为后续YOLO系列模型训练准备标准数据集。

本次任务核心目标：

  • 掌握LabelMe标注工具的安装与基本使用
  • 完成显示屏等目标的矩形标注
  • 规范标签命名与文件存储
  • 输出可直接用于训练的标注文件

二、环境准备与工具安装

本次使用 LabelMe 进行标注，它支持多边形、矩形、点、线等多种标注形式，适合小目标、不规则目标的精细标注。

安装命令（Miniconda虚拟环境终端执行）：

pip install labelme

安装完成后，直接在终端输入 labelme 即可打开可视化页面

三、数据集图像准备

本次标注目标为显示屏’键盘、鼠标、手柄，网上收集清晰、光照均匀、角度丰富图片，保证目标完整、无大面积遮挡，便于后续模型学习特征。

图片统一存放于文件夹，命名规范，便于管理与追溯。

四、LabelMe标注全过程

4.1 打开标注工具

终端运行：

labelme

打开后选择图片目录，进入标注主界面，界面包含图像区、标注区、文件列表区。

4.2 新建矩形标注

围绕电子设备取两点，确保框住整个电子设备，没有多余的空白。

标注完成后弹出标签输入框，统一标签命名，保证全数据集标签一致，避免训练出错。

4.3 保存标注文件

每标注完一张图，点击 Save，LabelMe 会自动生成与图片同名的 .json 标注文件，包含坐标、标签、图像尺寸等信息，是后续格式转换的核心文件。

六、任务总结与注意事项

本次任务完整完成了从环境配置、图像准备、工具使用到标注输出的全流程实践，掌握了LabelMe标注的核心操作，理解了目标检测数据集标注的规范与重要性。

重点注意事项：

  1. 标注轮廓尽量贴合目标边缘，提高数据集质量
  2. 标签名全程统一，不要随意修改
  3. 图片与json文件必须同名、同目录
  4. 标注完成后检查文件完整性，避免漏保存

通过本次任务，已具备独立制作目标检测标注数据集的能力，虽然中间遇到很多问题，比如我一开始准备使用Labelimg进行标注，但因为兼容问题，无法稳定使用，之后决定采用LabelMe.LabelMe还可以多边形标注，标注后的训练结果正确率较矩形标注正确率更高
