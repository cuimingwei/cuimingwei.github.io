任务二：Hexo + GitHub 个人博客搭建全记录

Created2026-02-03|Updated2026-02-05|技术实践
|Post Views:

一、任务背景

这次任务的目标是使用 Hexo 框架搭建个人博客，并将其部署到 GitHub Pages。同时，我需要为博客挑选主题，并把任务一和任务二的内容写成文章发布，最后确保能通过 GitHub 提供的地址正常访问。

二、环境准备

  1. 安装 Node.js 与 Git
    Hexo 是基于 Node.js 的静态博客框架，而 Git 用于将代码推送到 GitHub。我先在官网下载并安装了最新版的 Node.js 和 Git，安装完成后在命令行里用 node -v 和 git --version 确认安装成功。

  2. 配置 Git 与 GitHub 连接
    为了能把本地代码推送到 GitHub，我用 git config --global user.name "你的GitHub用户名" 和 git config --global user.email "你的GitHub邮箱" 配置了 Git 身份，然后生成了 SSH 密钥并添加到 GitHub 账户中，这样就可以免密推送代码了。

三、Hexo 博客搭建

  1. 安装 Hexo 脚手架
    在命令行执行：
    npm install -g hexo-cli
    这会把 Hexo 的命令行工具安装到全局，方便后续操作。

  2. 初始化博客项目
    我选了一个空文件夹作为博客根目录，在里面执行：
    hexo init my-blog
    cd my-blog
    npm install
    执行完后，my-blog 文件夹里就生成了 Hexo 的基础项目结构，包括配置文件、主题目录和文章文件夹。

  3. 本地启动预览
    执行 hexo s，然后在浏览器访问 http://localhost:4000，就能看到默认主题的博客页面了，这说明本地搭建成功。

四、主题选择与配置

Hexo 有很多开源主题可以选择，我最终选了简洁美观的 Butterfly 主题，步骤如下：

  1. 下载主题
    在博客根目录执行：

    git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

  2. 启用主题
    打开根目录的 _config.yml，找到 theme 字段，把值改成 butterfly。

五、部署到 GitHub Pages

  1. 创建 GitHub 仓库
    我在 GitHub 上创建了一个名为 用户名.github.io 的公开仓库，这是 GitHub Pages 的专属命名规则，只有这样才能通过 https://用户名.github.io 访问博客。

  2. 安装部署插件
    执行：

    npm install hexo-deployer-git –save

  3. 配置部署信息
    在根目录的 _config.yml 里找到 deploy 字段，修改为：

    deploy:
    type: git
    repo: git@github.com:你的用户名/你的用户名.github.io.git
    branch: main

  4. 生成并推送静态文件
    执行 hexo clean && hexo g && hexo d，这条命令会先清理旧的静态文件，再生成新的静态页面，最后推送到 GitHub 仓库。等待几分钟后，访问 https://用户名.github.io 就能看到线上的博客了。

六、任务内容发布

我把任务一和任务二的操作过程整理成了 Markdown 文章，放在 source/_posts 文件夹里,写完后重新执行 hexo g && hexo d，文章就同步到线上了。

七、加分项尝试

  1. 文章美化与图片优化
    我在 Markdown 里嵌入了 HTML 和 CSS 来调整排版，还把图片上传到了腾讯云 COS 对象存储，生成了公开的访问链接，替换了本地图片路径，这样解决了图片加载慢的问题。

  2. 添加返回顶部按钮
    我在主题的 scripts 目录里添加了自定义的 JS 文件，实现了点击右下角箭头按钮平滑滚动到顶部的动态效果，让博客的交互体验更好。

八、总结

通过这次任务，我不仅熟悉了 Hexo + GitHub Pages 的博客搭建流程，还学会了主题配置、本地调试和线上部署的完整链路。虽然中间遇到了 GitHub 访问慢、部署命令报错等问题，但借助AI并跟着慢慢调整，都解决了。
